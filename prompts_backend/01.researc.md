Я хочу создать Telegram-бота с RAG базой знаний для малого бизнеса.

Задача бота:
- Отвечать на вопросы клиентов по базе знаний о компании
- Собирать заявки (имя, телефон, вопрос) и уведомлять владельца
- Работать 24/7 без сервера (локальный запуск для старта)

Проведи исследование: какие технологии и подходы 
существуют для реализации в 2025-2026?

Покажи варианты по каждому компоненту:
1. Язык и фреймворк для Telegram-бота
2. Хранение и поиск по базе знаний (векторные БД и альтернативы)
3. LLM для генерации ответов
4. Оркестрация RAG pipeline

Для каждого варианта: плюсы, минусы, сложность освоения.
Финальная рекомендация с объяснением — почему именно этот стек
для команды с разным уровнем опыта. Я понимаю что RAG это усложнение для моей задачи, но я хочу сделать именно так.
