# Финальный пакет для разработки TVM.ru

**Цель:** Конвертировать посетителя в звонок/заказ/визит в офис  
**Платформа:** Мегагрупп CMS  
**Время реализации:** 1–2 дня

---

## 1. Итоговый порядок блоков

1. `header` — Хедер с часами работы и телефоном  
2. `hero` — Главный заголовок + УТП + CTA  
3. `directions` — 6 направлений (карточки с иконками)  
4. `problem-solution` — Боли → Решения (таблица сравнения с конкурентами)  
5. `proof` — Отзывы (реальные с Яндекс.Карт)  
6. `location` — Как добраться + парковка  
7. `faq` — FAQ (топ-5 вопросов)  
8. `guarantees` — Гарантии (только новый товар, возврат по чеку)  
9. `cta` — Форма заявки (телефон/тариф/билет)  
10. `footer` — Футер с картой и контактами

## 4. Маппинг "блок -> компонент"

| Блок ID | Название | Компоненты | Цветовая кодировка |
|---------|----------|------------|-------------------|
| `header` | Хедер | `section`, `button--service` | `--color-primary` |
| `hero` | Главный заголовок | `section`, `card`, `button--primary`, `button--service` | `--color-bg` |
| `directions` | 6 направлений | `section`, `card` (x6), `button--primary` | `--color-bg-alt` |
| `problem-solution` | Боли → Решения | `section`, `table`, `badge--success` | `--color-bg` |
| `proof` | Отзывы | `section`, `testimonial` (x3), `button--primary` | `--color-bg-alt` |
| `location` | Локация | `section`, `card`, `button--primary` | `--color-bg` |
| `faq` | FAQ | `section`, `accordion`, `button--primary` | `--color-bg-alt` |
| `guarantees` | Гарантии | `section`, `card`, `badge--success`, `button--primary` | `--color-bg` |
| `cta` | Форма заявки | `section`, `form`, `input`, `button--service` | `--color-bg-alt` |
| `footer` | Футер | `section`, `button--service` | `--color-bg` |

**Примечание:**  
- Каждый блок использует `section` с `padding: var(--space-2xl) 0`  
- Карточки направлений — цветовая кодировка из дизайн-системы (телефоны — `--color-phones`, билеты — `--color-aviatickets` и т.д.)  
- Акцентные CTA-кнопки — `--color-primary` или `--color-service`

## 5. Implementation checklist

- [ ] Создать CSS variables (цвета, типографика, spacing, радиусы, тени)  
- [ ] Сверстать хедер с фиксированным хедером (часы + телефон)  
- [ ] Сверстать hero-блок с заголовком, подзаголовком и двумя CTA-кнопками  
- [ ] Сверстать блок направлений (6 карточек с иконками и цветовой кодировкой)  
- [ ] Сверстать таблицу сравнения (блок болей → решений)  
- [ ] Сверстать блок отзывов (3 карточки с аватарами и звёздами)  
- [ ] Сверстать блок локации (карта с маркером + иконки транспорта)  
- [ ] Сверстать FAQ (аккордеон на JS или CSS-only)  
- [ ] Сверстать блок гарантий (3 галочки с текстом)  
- [ ] Сверстать форму заявки (имя, телефон, направление, комментарий)  
- [ ] Сверстать футер с картой и контактами  
- [ ] Добавить адаптивность (мобильная версия)  
- [ ] Проверить загрузку (LCP < 2.5s, FID < 100ms)  
- [ ] Добавить микродоверие (рейтинг 4.7/5, 40+ отзывов)  
- [ ] Протестировать формы (валидация, отправка)

## 6. Критерии готовности (acceptance criteria)

- [ ] Все 10 блоков присутствуют в правильном порядке  
- [ ] Хедер содержит часы работы и телефон (крупно, в шапке)  
- [ ] Hero-блок содержит УТП и двойной CTA («Оставить заявку» + «Позвонить»)  
- [ ] Блок направлений показывает все 6 направлений с цветовой кодировкой  
- [ ] Таблица сравнения содержит 6 строк (боли → решения)  
- [ ] Отзывы — 3 реальных цитаты с датами и звёздами  
- [ ] Локация содержит адрес, метро, автобус, парковку  
- [ ] FAQ — 5 вопросов с ответами (аккордеон)  
- [ ] Гарантии — 3 пункта (только новый товар, возврат по чеку, помощь)  
- [ ] Форма заявки содержит 4 поля (имя, телефон, направление, комментарий)  
- [ ] Футер содержит адрес, часы, телефон, email, 5 направлений  
- [ ] Адаптивность: мобильная версия (768px и ниже)  
- [ ] Кнопки на мобильных — на всю ширину  
- [ ] Загрузка: LCP < 2.5s, FID < 100ms  
- [ ] Формы валидируются и отправляются (без ошибок)
