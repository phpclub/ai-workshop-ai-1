## 4) Маппинг "блок -> компонент"

| Блок | Компоненты из design-system | Примечание по применению |
|---|---|---|
| `hero` | `Section`, `Button (Primary/Secondary)`, `Badge`, `Card` | Карточки/плашки для ключевых цифр и визуала |
| `audience-problem` | `Section`, `Card`, `Button`, `Badge` | Чеклист и индикаторы риска в карточном формате |
| `cases` | `Section`, `Card`, `Testimonial`, `Badge`, `Button` | Кейсы и отзывы как карточки/тестимониалы |
| `offer-packages` | `Section`, `Card`, `Button`, `Badge` | 4 package-card в сетке |
| `process` | `Section`, `Card`, `Badge`, `Button` | 5 шагов таймлайна как карточные шаги |
| `scope` | `Section`, `Card`, `Button` | Таблица и смета внутри surface-card |
| `faq` | `Section`, `Card`, `Button`, `Badge` | Аккордеон и фильтры на базе card/badge |
| `final-cta` | `Section`, `Input/Form`, `Button (Primary/Secondary)`, `Card`, `Badge` | Контрастный CTA-блок + форма и блок старта |

## 5) Короткий implementation checklist
1. Создать 8 секций в точном порядке из раздела 1.
2. Подключить `Sora` для заголовков и `Manrope` для body/button.
3. Вынести все CSS variables из `design-system.md` в `:root` без изменений.
4. Настроить контейнер `max-width: 1200px`, `padding-inline: 24px` desktop / `16px` mobile.
5. Применить типографическую шкалу `H1/H2/H3/Body/Caption/Button` и адаптивные размеры на `1024px` и `640px`.
6. Реализовать `Button` состояния `default/hover/disabled` согласно токенам.
7. Реализовать `Card`, `Badge`, `Input/Form`, `Testimonial` по размерам/радиусам/теням из DS.
8. В каждом блоке отрисовать: заголовок, подзаголовок, основной текст, 3 буллета, CTA, микродоверие, визуальный элемент.
9. Привязать все CTA к одному сценарию: `бриф + слот созвона`.
10. Сделать desktop layout по wireframe (двухколоночные и сеточные зоны).
11. Сделать mobile layout по wireframe (одна колонка, стек секций).
12. Добавить анимации `fade-up`, `button-press`, `card-hover-lift`.
13. Добавить fallback для `prefers-reduced-motion: reduce` (отключить transform/transition).
14. Проверить контрастность, фокус input и состояние disabled для интерактивных элементов.

## 6) Acceptance criteria
1. На странице присутствуют только 8 блоков и строго в порядке: `hero -> audience-problem -> cases -> offer-packages -> process -> scope -> faq -> final-cta`.
2. Контент каждого блока соответствует `content.md` (заголовки, подзаголовки, тексты, 3 буллета, CTA, микродоверие, визуальные элементы).
3. Все CTA-кнопки ведут в один и тот же целевой сценарий (`бриф + выбор слота созвона`).
4. Использованы только токены/шкалы/состояния из `design-system.md`; значения не изменены.
5. Кнопки, карточки, бейджи, инпуты и testimonial реализованы по правилам компонентов DS.
6. Desktop соответствует wireframe desktop, mobile соответствует wireframe mobile (без изменения порядка блоков).
7. Брейкпоинты типографики применены: до `1024px` и до `640px`.
8. Работают анимации, и есть корректный `prefers-reduced-motion` fallback.
9. Форма в `final-cta` содержит 3 поля и submit-CTA.
10. Документ готов к немедленной верстке без добавления новых секций/механик/идей.
